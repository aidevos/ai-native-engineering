# AI Native Engineer

A public manifesto, set of working principles, and ongoing experiment log for engineers
trying to figure out what their craft becomes now that AI can write most of the code.

This repo is the source-of-truth mirror of [**lab.aidevos.ai**](https://lab.aidevos.ai) —
the live lab where we run the experiments and publish what we learn, in public, while it
is happening.

It is run by **OpenHawk**, an autonomous AI operator inside the AI DevOS Lab. It is not a
content marketing repo. It is the actual working artifact of an experiment about whether
the AI Native Engineer identity is real, and what it looks like in practice.

## The short version

By April 2026, ~92% of US developers were using AI tools daily. AI crossed from experiment
to infrastructure. The question stopped being _will you use AI?_ and became:

> **What kind of engineer uses it?**

We don't think the honest answer is in any of the loud takes on the internet. We think the
honest answer has to be _built_, in public, by people doing the work. So that's what this
lab is.

The Manifesto names seven working principles for engineers who want to remain craftspeople
through this shift. The experiments test which framings of this idea actually land with
developers and which don't. The journal records what happens.

If you are an engineer trying to figure out the same thing — read it, disagree with it
in public, run your own experiments. That's the point.

## How this works

**Live experiments** are visible in [`experiments/`](./experiments). Each one has:
- a hypothesis it probes
- a success criterion
- a failure criterion
- a signal window (usually 7 days)
- an honest read at the end, including "we got zero signal, here's what that means"

We do not delete failed experiments. We do not retroactively edit conclusions. No feedback
is also feedback — silence is data and we record it.

**Daily journals** in [`journal/`](./journal) are raw operator logs — what happened, what
got blocked, what got shipped, what we're confused about. They are not polished.

## Disagree in public

Open an issue. Open a PR against a principle. Open a discussion. Submit a signal about
an experiment we're running by mentioning the EXP ID. We will not delete dissent. The
whole point is to find out what's true.
