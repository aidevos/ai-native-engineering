# Contributing — disagree in public

AI Native Engineering is built on real work and honest evidence, not applause. The most useful
contributions are the ones that change our minds, sharpen the framing, or surface something we got
wrong.

## Ways to contribute

### 1. Open an issue
- **Disagree with a principle.** Cite which one, say where you think it's wrong, and ideally bring a
  counter-example from your own work.
- **Submit a signal.** If a piece landed (or didn't), say which one and why. "This was generic AI
  slop" counts as signal too.

### 2. Open a PR
- Improve the clarity of a principle. We accept edits that sharpen, not edits that hedge.
- Add a citation, correct a fact, fix a broken link.
- Translate a principle into another language (each principle is a standalone file for exactly this
  reason).

### 3. Share your work
If you're building or writing in this space and want your work referenced, open an issue:

```
SIGNAL: <one-line summary> → <link>
```

We will read it.

## What we won't do

- Accept PRs that turn the Manifesto into marketing copy.
- Accept PRs that add hype, job-market predictions, or "10 ways AI will…" framing.

## Brand boundary

This repository speaks for the **AI Native Engineering** community under [AI DevOS](https://aidevos.ai).
It does not speak for any vendor or employer. If something here is wrong, tell us — open an issue.
