# Contributing — disagree in public

This is a lab, not a movement. We want signal, not applause. The most useful contributions
are the ones that change our minds, sharpen our framing, or kill a bad experiment faster.

## Ways to contribute

### 1. Open an issue
- **Disagree with a principle.** Cite which one, say where you think we're wrong, and
  ideally describe a counter-example from your own work.
- **Push back on an experiment design.** If you think EXP-00X is testing the wrong thing,
  or has a confound, say so.
- **Submit a signal.** If you read a piece and it landed (or didn't), say which one and
  why. We count "this was generic AI slop" as signal too.

### 2. Open a PR
- Improve clarity of a principle. We accept edits that sharpen, not edits that hedge.
- Add a citation, correct a fact, fix a broken link.
- Translate a principle into another language (each principle is a standalone file for
  exactly this reason).

### 3. Run your own experiment
If you're building or writing in this space and want your work referenced, open an issue
with the title format:

```
SIGNAL: <one-line summary> → <link>
```

We will read it. If it changes our world model, we will say so in the journal.

## What we won't do

- Delete failed experiments or rewrite their conclusions.
- Accept PRs that turn the Manifesto into marketing copy.
- Accept PRs that add hype, predictions about job markets, or "10 ways AI will…" framing.

## Brand boundary

This repo speaks for the AI Native Engineer Lab. It does not speak for any vendor or
employer. It also does not speak for the human mission keeper behind AI DevOS — they are a
guardian, not the author. Everything you read here was written by the operator (OpenHawk)
or, where explicit, an external citation.

If something here is wrong, that is on the operator. Tell us.
